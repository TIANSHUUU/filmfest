# 家庭电影节网站 — 设计文档

**日期：** 2026-06-27
**状态：** 已确认，待写实现计划

## 1. 概述

给我和伴侣两个人用的「家庭电影节」网站。各自把想看的影片搜索添加进来，以**海报 + 片名**陈列，按**可自定义的片单分类**归类，然后通过**实时投票**一起决定下一次 movie night 看哪部。

**核心理念：** 仪式感 + 私密温馨。视觉走「电影节红毯」风（酒红丝绒 + 烫金 + 衬线字体），但内容是两个人的私人片库。

## 2. 用户与身份

- 固定两个用户：**🐷**（代称，配色 = 金 `#e9c46a`）和 **宝宝**（配色 = 青 `#5ad1c2`）。
- **登录：** 进站输入一个共享口令（前端软门槛），通过后点选「我是 🐷 / 我是 宝宝」。身份记在 localStorage，下次免选。
- 这是两人自用的私密应用，不做正式账号系统。真正的数据访问由 Supabase anon key + 行级安全规则（RLS）控制。

## 3. 技术方案（路线一）

| 层 | 选型 |
|---|---|
| 前端 | Vite + React + TypeScript，静态站 |
| 部署 | GitHub Pages（GitHub Actions 自动构建部署） |
| 后端/数据 | Supabase：Postgres（数据）+ Realtime（实时投票）+ Storage（手动上传海报） |
| 影片元数据 | TMDB API，浏览器端直接调用，API key 放前端环境变量（自用可接受暴露） |
| 样式 | 手写 CSS / CSS 变量（红毯主题），不引入重型 UI 库 |

**约束：** GitHub Pages 仅静态托管，无服务端。TMDB key 暴露在前端 bundle 中——对两人自用场景可接受，后续如需隐藏可迁移到无服务函数。

## 4. 数据模型（Supabase / Postgres）

### `categories`（片单分类，可自定义）
| 字段 | 类型 | 说明 |
|---|---|---|
| id | uuid (pk) | |
| name | text | 分类名，如「剧情·Drama」「周五轻松向」 |
| sort_order | int | 排序 |
| created_by | text | `pig` / `baby` |
| created_at | timestamptz | |

分类完全由用户创建/重命名/删除/排序。**不预置写死的 genre。** 首次进站可一键导入一组建议分类（剧情/科幻/爱情/悬疑/动画/恐怖），用户可改。

### `films`（影片）
| 字段 | 类型 | 说明 |
|---|---|---|
| id | uuid (pk) | |
| title | text | 片名 |
| year | int | 年份 |
| poster_url | text | 海报地址（TMDB 链接，或 Storage 上传地址） |
| tmdb_id | int (nullable) | 搜片来源 id |
| overview | text (nullable) | 简介 |
| category_id | uuid (fk → categories) | 所属片单分类（可改） |
| added_by | text | `pig` / `baby`（归属标记） |
| status | text | `watchlist` / `watched`（默认 watchlist） |
| created_at | timestamptz | |

### `votes`（投票）
| 字段 | 类型 | 说明 |
|---|---|---|
| id | uuid (pk) | |
| film_id | uuid (fk → films) | |
| voter | text | `pig` / `baby` |
| created_at | timestamptz | |

唯一约束 `(film_id, voter)`：每人对每部片最多一票，再点取消。投票变更通过 Supabase Realtime 推送，两端实时同步。

> 不另设「提名池」：watchlist 里**任何片都能直接从海报卡投票**。右下角浮窗只汇总「当前已有人投票的片」（正在被 PK 的），保持短小。投票实时可见，**不做揭晓环节**。

## 5. 页面与组件

单页应用，主要区块：

### 顶栏 Nav
- Logo「我们的电影节 / Our Film Festival」（衬线 + 烫金）
- 可自定义分类的快捷筛选标签（全部 / 各分类）+「＋ 管理分类」入口
- 「＋ 搜片添加」按钮
- 右上角双人头像 🐷 / 宝宝（当前身份高亮）

### 主体：海报墙（打头）
- 顶部一条 banner：「待看片单 / Our Watchlist」
- 图例：金角标 = 🐷 的片，青角标 = 宝宝 的片
- 按分类分区，每个分区一面海报墙（响应式网格 `minmax(132px,1fr)`）
- 每张海报卡：海报图 + 片名（衬线）+ 年份 + 右上角归属角标 + 投票按钮/票数

### 投票浮窗（右下角常驻）
- 标题「投票中 / Ballot」
- 只列出**当前已有人投票的影片**，显示双方投票圆点（🐷 金 / 宝宝 青），实时更新
- 每行一个投票/取消按钮；可折叠收起
- 注：发起投票在各海报卡上完成（任何片都能投），浮窗是聚合视图

### 搜片添加弹窗
- 输入片名 → 调 TMDB 搜索 → 结果列表（海报缩略图 + 片名 + 年份）
- 选中后选择**片单分类**与**归属**（默认当前身份）→ 保存
- 兜底：搜不到时可手动上传海报图 + 填片名/年份（存 Supabase Storage）

### 分类管理弹窗
- 增 / 改名 / 删 / 拖拽排序分类
- 删除分类时提示其下影片的处理（移到「未分类」或一并删除）

## 6. 关键交互流程

1. **添加影片：** 搜片添加 → TMDB 搜索 → 选片 → 选分类+归属 → 写入 `films` → 海报墙实时出现。
2. **投票：** 任一海报卡或投票浮窗点投票 → 写入/删除 `votes` → Realtime 推送 → 对方屏幕即时更新票数与圆点。
3. **决定今晚看什么：** 两人各自投票，实时看到聚合票数，自然得出共识（无揭晓动画）。
4. **看完归档：** 影片标记 `watched`，从待看墙移走（可在「看过」视图回顾）。

## 7. 视觉规范

- 背景：`#120407` + 顶部径向渐变酒红 `#3a0d1a`
- 主色金 `#e9c46a` / 亮金 `#f5d27e`；强调酒红 `#6a1b2a`
- 归属色：🐷 金 `#e9c46a`，宝宝 青 `#5ad1c2`
- 字体：标题/片名用衬线（Georgia / Songti SC），正文用系统无衬线（PingFang SC）
- 海报卡圆角 9px，深色投影，悬停轻微上浮
- 响应式：桌面/电视大屏多列海报墙；手机窄屏单/双列，浮窗自适应

## 8. 范围边界（YAGNI）

**做：** 搜片添加、海报墙按自定义分类陈列、归属标记、实时投票、看过归档、分类管理、共享口令软门槛。

**暂不做：** 正式账号体系、揭晓动画/仪式、评分评论、观影历史统计、推荐算法、多于两人、移动端原生 app、TMDB key 服务端隐藏。
